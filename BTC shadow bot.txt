📌 Strategy Overview
Market: Polymarket — "BTC Up or Down - 5 Minutes" Core Logic: In the final 60–90 seconds of a 5-minute market, the contract price lags behind where Bitcoin is actually trading. The bot identifies this lag and fires a trade milliseconds before the market corrects/settles.
Manual version (from your reel):
	•	If "Buy Up" or "Buy Down" drops below 25¢ in the first 2 minutes → buy it
	•	Immediately set a limit sell at 40¢ → ~2x return
Automated version (Clawbot logic):
	•	Watch thousands of markets simultaneously
	•	Detect price lag vs. real BTC spot price
	•	Fire trades in milliseconds before correction
	•	Repeat at high volume for compounding small edges

🤖 Bot Architecture
1. Data Feeds Required
- Polymarket API (market prices, order book, market close time)
- Real-time BTC spot price feed (Binance or Coinbase WebSocket)
- Polymarket CLOB (Central Limit Order Book) API for order placement
2. Core Logic Flow
EVERY 5-MINUTE MARKET CYCLE:
1. Fetch current BTC spot price (Binance WS)
2. Fetch Polymarket contract prices for that market
3. Calculate: Is BTC already UP or DOWN vs. market open price?
4. Check: Is the contract price lagging? (e.g., BTC is up but "Up" contract is still at 0.45 instead of 0.80+)
5. If lag detected AND < 90 seconds to close → FIRE BUY
6. Set limit sell at target price OR hold to settlement
7. Log trade, P&L, and outcome
3. Lag Detection Formula
btc_direction = "UP" if current_btc > market_open_btc else "DOWN"
contract_price = get_polymarket_price(direction=btc_direction)
lag_threshold = 0.15  # if contract is more than 15 cents underpriced

if contract_price < (expected_fair_value - lag_threshold):
    fire_trade(direction=btc_direction, size=position_size)

🔑 Polymarket API Details
Base URL: https://clob.polymarket.com
Auth: L1 or L2 Ethereum wallet signature
SDK: pip install py-clob-client

Key endpoints:
- GET /markets → list active markets
- GET /orderbook/{market_id} → live order book
- POST /order → place buy/sell order
- GET /positions → current holdings
Wallet setup:
	•	Create an Ethereum wallet (MetaMask or raw private key)
	•	Fund with USDC on Polygon network
	•	Bot signs all orders with private key

🌏 Australia Geo-Block Workaround
You have two solid options:

✅ Option 1 — Residential Proxy (Recommended)
Route all bot traffic through a US-based residential proxy.
# In your requests/httpx config:
proxies = {
    "http": "http://user:pass@us-residential-proxy:port",
    "https": "http://user:pass@us-residential-proxy:port"
}
Best providers:
	•	Bright Data (most reliable, residential IPs)
	•	Oxylabs
	•	Smartproxy
✅ Looks like real US traffic. Low ban risk. Works with WebSockets.

✅ Option 2 — VPS in the US (Most Reliable for a Bot)
Deploy the entire bot on a US-based cloud server — no geo-block at all since the server is in the US.
Providers:
- DigitalOcean (New York / San Francisco droplet) ~$6/mo
- Vultr
- AWS EC2 (us-east-1)
- Hetzner (US locations)
This is the cleanest solution — your bot runs 24/7 on a US IP, no VPN overhead, no latency added, no risk of proxy dropping mid-trade.

⚠️ Option 3 — VPN on Local Machine (Not Recommended for Bot)
A regular VPN (NordVPN, ExpressVPN) works for manual browsing but is unreliable for a trading bot because:
	•	VPN can disconnect mid-session
	•	Adds latency (bad for millisecond execution)
	•	IP can get flagged

⚡ Latency Optimisation (Critical)
Since this is a millisecond-timing strategy:
1. Host bot on US East VPS (closest to Polymarket servers)
2. Use WebSocket connections (not REST polling) for price feeds
3. Pre-authenticate and keep session alive
4. Pre-calculate position sizes so no compute delay at fire time
5. Use async/await throughout (never blocking calls)
